    • Dynamic Links
    • Setting up Dynamic Content and Dynamic Links.

      Movable Ink technology allows for dynamic content in the images of your emails. This feature becomes really powerful when matched with Movable Ink's dynamic linking capability. Dynamic links mean that when your recipient clicks on an MI element in an email, they'll be directed to a landing page based on what you see in the ever-changing image.

      This is useful if you want to showcase a section on your website that frequently updates. The dynamic cropping allows you to showcase the latest piece of content. The dynamic linking allows you to direct clickers to landing pages specific to the content they saw.

      On this example page, you are shown the latest news story from a fake news site. This page updates to show a new headline and link every hour (look at the source for the JS that drives this).

      When Movable Ink crops this page, it also stores the #dynamic_link HREF in memory. If the email recipient clicks on the image, MI will retrieve this memorized HREF and 302 the recipient to this URL.

    • Animation
    • A Look at How Animation Works

      This page features code from the countdown timer app in the Movable Ink interface (with a few modifications). It is programmed to always show the amount of time between now and 3pm, five calendar days in the future.

      The Movable Ink crop that is programmed to crop this page also has the animation feature checked. That means the MI server will visit this page, take a snapshot, execute the "manualAdvance()" function, take another shot, execute "manualAdvance()" again... and do this 60 times. Once all 60 snapshots are taken, the server will compose an animated gif and deliver that to the email recipient.

      If you are creating a crop with an animation, be mindful of how much data you're trying to pack in an animated gif. A 500px by 400px image at 60 frames is functionally 12 megapixels of data. This will probably take too long for an email, as the MI server must snapshot and assemble all this data before it can initiate the delivery of a potentially large file.

    • Requesting Meta-Fata
    • An Overview

      When an email client requests images from Movable Ink, it will pass along information that may be useful in determining what to show. These are things like user agent and IP address. Movable Ink interprets these two points of data to determine device type and an estimated location of open.

      Gotchas

      Movable Ink will make a separate crop for every set of query parameters. If you enable "pass location", you should expect slower image load times because each image request will be unique and we will not be able to retrieve information from a cache.
